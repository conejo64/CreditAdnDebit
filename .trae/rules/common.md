# 前端企业级强制编码规范
## 一、通用基础强制规则
1. 编码统一 UTF-8，缩进固定2空格，文件末尾保留单行空行，删除尾随空格、多余空行。
2. 禁止使用 var，统一使用 const/let；禁止隐式全局变量、随意挂载全局属性。
3. 禁止魔法数字、魔法字符串，固定业务常量统一抽离至 constants 目录维护。
4. 所有异步逻辑强制使用 async/await，禁止回调嵌套，异步请求必须 try/catch 捕获异常。
5. 禁止提交 debugger、console.log、无效注释、废弃代码块、临时测试代码。
6. 代码逻辑嵌套层级不允许超过3层，超三层必须抽离函数、反向条件、提前return优化。
7. 通用工具函数、复用逻辑统一抽入 utils、hooks，保持纯函数、单一职责。
8. 所有业务代码必须保证可读性：语义化命名、必要注释、逻辑分段、禁止堆砌式编码。

## 二、命名强制规范
1. 文件夹、文件名、静态资源、CSS类名：kebab-case。
2. 页面组件、业务组件、TS类、枚举：PascalCase。
3. 变量、函数、方法、实例、自定义钩子：camelCase。
4. 全局常量、业务枚举值、环境变量：UPPER_SNAKE_CASE。
5. 禁止中文、拼音、无意义缩写、单字母变量、模糊命名，所有命名必须见名知意。
6. 布尔值变量统一语义化：isXXX / hasXXX / shouldXXX / enableXXX。

## 三、组件化 & 拆分 & 单文件行数【核心强约束】
1. **严格禁止单文件大代码**
   - 纯组件/页面文件：**最大行数 ≤ 350行**
   - 工具类、类型文件：最大行数 ≤ 500行
   - 超出必须强制拆分，禁止堆砌所有逻辑在一个页面/组件内。
2. **单一职责原则（强制）**
   - 一个组件只负责一件事，禁止大杂烩组件。
   - 页面 = 容器组件，只负责：布局、数据聚合、状态分发、子组件组合。
   - 页面内部业务区块、表单、弹窗、列表项、操作区，必须拆分为独立子组件。
3. **逻辑分层拆分强制要求**
   - 复杂业务逻辑、计算逻辑、数据处理、格式化、筛选过滤：禁止写在视图层。
   - 统一抽离至：hooks / utils / service / 独立组合式函数。
   - 接口请求、副作用、数据获取逻辑，禁止直接写在组件顶层或视图内部。
4. **复用逻辑强制抽离**
   - 重复代码出现2次及以上，必须抽离公共组件/公共钩子/工具方法。
   - 列表渲染项、通用卡片、通用表单控件、通用弹窗，统一沉淀公共组件。
5. **目录内聚管理**
   - 复杂页面建立独立文件夹，拆分：index 入口、子组件、局部样式、局部类型、局部钩子。
   - 禁止多个无关业务逻辑耦合在同一个文件。

## 四、可读性 & 可维护性强制规范
1. 复杂逻辑、特殊业务规则、临时兼容逻辑，必须添加**业务注释**说明目的。
2. 代码按逻辑区块分段：数据定义 → 状态声明 → 计算处理 → 副作用 → 事件方法 → 视图渲染。
3. 禁止超长一行代码，链式调用、复杂三元、复杂对象格式化换行书写。
4. 禁止复杂嵌套三元表达式，多条件判断抽离为函数/枚举/配置项。
5. 配置型、映射型、字典型数据，统一抽离为常量配置，禁止硬编码判断。
6. 条件判断语义化，禁止多层 && 短路写法滥用，复杂条件封装为语义化布尔变量。

## 五、TypeScript 强制规则
1. 业务代码严格禁用 any、unknown，必须使用 interface / type 定义结构化类型。
2. 接口返回、请求参数、组件Props、全局状态、本地临时数据必须完整类型约束。
3. 公共通用类型统一存放 types 目录，局部独有类型放在当前文件顶部或同目录 type 文件。
4. 固定选项、状态码、字典项、业务枚举，优先使用 enum / 常量枚举管理。
5. 禁止滥用类型断言 as，类型缺失时优先补全类型定义而非强制覆盖。
6. 组件Props 必须完整定义，必填/可选默认值严格区分。

## 六、模板 & HTML 规范
1. 优先使用语义化标签 header / main / section / article / footer / nav。
2. 标签属性统一双引号，标签嵌套规范，合理闭合，保持结构整洁。
3. 图片必须配置 alt 属性，交互元素补充 aria 无障碍属性。
4. 禁止模板内大量复杂表达式、长计算、复杂函数执行，所有复杂逻辑抽离脚本。
5. 自定义属性统一使用 data-* 前缀，禁止自定义原生DOM属性。

## 七、样式 CSS/SCSS 规范
1. 样式类名统一 kebab-case，语义化命名，禁止拼音与无意义命名。
2. SCSS/LESS 嵌套层级不超过3层，杜绝深层嵌套污染全局样式。
3. 颜色、圆角、间距、阴影、字号统一使用全局变量，禁止硬编码视觉值。
4. 非极端业务场景禁止使用 !important，特殊使用必须添加注释说明原因。
5. 隔离样式污染，Vue 使用 scoped，React 优先 CSS Modules / styled / 独立命名空间。
6. 禁止通配符选择器、低效层级选择器，保证页面渲染性能。

## 八、Vue 专项强制规范
1. SFC 文件固定顺序：script setup → template → style。
2. 组件 Props 必须声明类型、默认值、校验规则，禁止裸写props。
3. 严禁 v-if 与 v-for 写在同一标签，循环渲染必须绑定稳定唯一key。
4. 模板复杂计算逻辑统一使用 computed，业务处理逻辑抽离组合式函数。
5. 大页面按模块拆分子组件，弹窗、表单、表格、操作栏全部独立拆分。
6. 自定义事件必须通过 emits 显式声明，全局状态合理拆分模块化。

## 九、React 专项强制规范【新增补全】
1. 优先使用**函数组件 + Hooks**，禁止class组件业务新增开发。
2. 组件文件拆分：页面容器组件、纯UI子组件、业务逻辑hooks严格分离。
3. 所有组件 Props 必须使用 TS 类型定义，禁止匿名类型、禁止裸传参数。
4. 状态管理分层：
   - 局部状态：useState
   - 复杂本地状态：useReducer
   - 全局状态：模块化拆分，禁止全局状态泛滥。
5. 副作用规范：
   - useEffect 必须严格补齐依赖项，禁止空依赖滥用。
   - 订阅、定时器、事件监听、请求订阅，必须在 useEffect 清理函数销毁。
6. 列表渲染：
   - map 渲染必须使用稳定唯一 key。
   - 复杂列表项强制拆分为独立 Item 组件，禁止页面内长列表JSX堆砌。
7. 事件处理：
   - 复杂事件逻辑抽离独立函数，禁止行内写大量逻辑。
   - 频繁触发事件使用 useCallback 缓存，纯计算使用 useMemo。
8. 路由与页面：
   - 页面只做组合与调度，不写重型业务逻辑。
   - 弹窗、抽屉、表单组件统一抽离为独立组件或业务模块内组件。
9. 禁止巨型 JSX
   - 渲染区域过长必须拆分多个小型渲染函数 / 子组件。
   - 禁止在 return 内部写大量变量定义、数据处理、循环逻辑。

## 十、接口与请求规范
1. 所有网络请求统一全局封装，统一处理超时、token、拦截、错误提示。
2. 接口地址、请求方法、入参结构统一集中管理，禁止页面硬编码接口地址。
3. 重复请求、高频操作增加防抖节流，组件卸载时取消pending请求，防止内存泄漏。
4. 区分网络异常、业务码异常，全局统一兜底处理，页面只保留个性化业务逻辑。

## 十一、工程目录规范
1. 严格遵循分层目录：api / assets / components / constants / hooks / layouts / pages / router / store / types / utils。
2. 全局公共组件放置 components/common，业务专属组件就近目录内拆分存放。
3. 业务页面采用文件夹结构：index.tsx/index.vue + 子组件 + local-hooks + local-types。
4. 环境配置区分开发/测试/生产，敏感密钥、私有配置禁止硬编码提交。

## 十二、代码健壮性
1. 所有对象、数组、接口返回值增加空值防护，使用可选链、空合并、默认值兜底。
2. 定时器、DOM事件监听、订阅消息、第三方实例，组件销毁必须手动销毁清除。
3. 表单校验、用户输入内容做格式校验与特殊字符转义，预防XSS风险。
4. 路由鉴权、按钮权限、接口权限统一管控，前后端双向校验。

## 十三、Git 提交规范
1. 分支命名：feat/xxx、fix/xxx、hotfix/xxx、refactor/xxx、docs/xxx。
2. 提交信息格式：type(scope): 简短描述，语义清晰、无无效提交。
3. 禁止直接推送合并至主干分支，代码变更需自查代码行数、拆分、规范校验。
4. 严格忽略 node_modules、本地配置、缓存文件、环境密钥、日志文件。
